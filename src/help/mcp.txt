shard mcp — Model Context Protocol server

Usage: shard mcp

Starts an MCP server on stdio (JSON-RPC 2.0). Reads requests from stdin,
writes responses to stdout. All diagnostic output goes to stderr.

The MCP server connects to the daemon via IPC internally. If the daemon
is not already running, the MCP server will start it automatically as a
background process.

PROTOCOL
  initialize                  Server info + capabilities
  notifications/initialized   (no response)
  tools/list                  List available tools
  tools/call                  Execute a tool

AVAILABLE TOOLS
  shard_query             Search the knowledge base (key required)
                          - Omit shard: searches all shards via vector routing
                          - Set shard: searches one shard directly
                          - Set depth>0: follows cross-links (BFS graph traversal)
  shard_discover          List/filter shards from the registry (no key)
  shard_discover_refresh  Re-scan .shards/ directory and refresh registry (no key)
  shard_remember          Create a new shard with catalog and gates (no key)
  shard_catalog           Read a shard's catalog (no key)
  shard_gates             Read a shard's routing gates (no key)
  shard_list              List all thought IDs in a shard (no key)
  shard_status            Health check: name, thought count, uptime (no key)
  shard_read              Decrypt and read a thought by ID (key required)
  shard_write             Write a new encrypted thought (key required)
  shard_update            Update a thought's description/content (key required)
  shard_delete            Delete a thought by ID (key required)
  shard_dump              Export all thoughts as markdown (key required)

All shard-targeting tools take a 'shard' argument (the shard name).
Tools marked "key required" also need a 'key' argument (64-hex master key).

Start MCP:  shard mcp
The daemon is started automatically if not already running.
To start the daemon manually:  shard daemon
