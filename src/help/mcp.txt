shard mcp — Model Context Protocol server

Usage: shard mcp

Starts an MCP server on stdio (JSON-RPC 2.0). Reads requests from stdin,
writes responses to stdout. All diagnostic output goes to stderr.

The MCP server connects to the daemon via IPC internally. If the daemon
is not already running, the MCP server will start it automatically as a
background process.

PROTOCOL
  initialize                  Server info + capabilities
  notifications/initialized   (no response)
  tools/list                  List available tools
  tools/call                  Execute a tool

AVAILABLE TOOLS (8)
  shard_discover          What exists? No params = full TOC. shard = info card.
                          query = filter. refresh = re-scan disk. (no key)
  shard_query             Find thoughts by search. shard = direct lookup.
                          No shard = auto-route. depth > 0 = cross-shard BFS. (key required)
  shard_read              Read thought by ID. chain = revision history. (key required)
  shard_write             Store thought. No id = create. id = update.
                          revises = revision link. (key required)
  shard_delete            Delete thought by ID. (key required)
  shard_dump              Export shard as markdown. (key required)
  shard_remember          Create new shard with catalog and gates. (no key)
  shard_events            Read or emit events. shard = read mode.
                          source + event_type = emit mode. (no key)

All shard-targeting tools take a 'shard' argument (the shard name).
Tools marked "key required" also need a 'key' argument (64-hex master key).

Start MCP:  shard mcp
The daemon is started automatically if not already running.
To start the daemon manually:  shard daemon
