shard connect — open a session to a running shard

Usage: shard connect [name]

  <name>  Name of the shard to connect to (default: daemon)

Opens a persistent IPC connection to the named shard and reads frontmatter
messages from stdin. Each message is sent over the connection, the response
is printed to stdout. Repeats until EOF.

One connection for the entire session — no reconnect overhead per op.
This is the primary interface for AI agents interacting with shards.

Example:
  cat <<'EOF' | shard connect
  ---
  op: registry
  ---
  ---
  op: search
  name: notes
  key: <64-hex master key>
  query: weather
  ---
  ---
  op: read
  name: notes
  key: <64-hex master key>
  id: abc123
  ---
  EOF

Responses use the same frontmatter format. Parse the frontmatter
for status, ids, items, etc. The body (after closing ---) is the content.

Run 'shard help shard' for the full operation reference.
